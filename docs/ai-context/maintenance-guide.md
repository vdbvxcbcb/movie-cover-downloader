# AI 图谱维护指南

本目录要保持短、准、可维护。它服务于 AI 快速定位，不追求覆盖每个函数。

## 什么时候必须更新

修改以下内容时，同步更新 `docs/ai-context`：

- 新增、删除或重命名跨层入口，例如 Tauri command、runtimeBridge 方法、sidecar `MCD_COMMAND`。
- 改变搜索、自动下载、选图发现、选图下载、Cookie、持久化、图片处理任一主链路。
- 改变选图下载约束，例如分类、分页/游标、选择/预览、重复任务检测。
- 改变持久化字段、SQLite 表、状态快照结构。
- 移动关键文件或拆分热点文件。
- 新增真实支持的站点或下载来源。

## 什么时候不需要更新

这些改动通常不用更新图谱：

- 纯样式微调。
- 单个组件内部的小型 UI 文案调整。
- 测试用例补充但不改变行为。
- 局部变量重命名。
- 不改变跨层契约的内部实现优化。

## 更新原则

- 保持文档是导航，不把源码重新抄一遍。
- 优先描述“从哪里进、经过哪里、谁拥有状态、谁输出结果”。
- 图谱如果不确定，先读代码再更新，不要凭 README 或规划文档推断。
- 不要把未落地规划写成当前能力。
- Mermaid 图只画主干路径，不画所有错误分支。
- 模块索引只列 AI 常用入口和边界，不列每个普通组件。

## 推荐检查清单

改完图谱后检查：

- `project-map.md` 的速查表是否仍能把任务导向正确文件。
- `runtime-flows.md` 的链路是否包含新的跨层入口或事件。
- `module-index.md` 的文件路径是否真实存在。
- `AGENTS.md` 是否仍指向正确的 AI 图谱入口。
- 文档中没有把 Cookie、用户本地路径、下载图片或本地状态库写成示例真实值。

## 图谱和源码冲突时

以源码和测试为准。处理方式：

1. 先完成当前修复或改动。
2. 更新图谱中冲突的最小段落。
3. 在最终说明里指出图谱已同步。
